# RDS Terraform Module

This module creates an AWS RDS database instance for the project.

The current implementation is designed for a cost-optimized development environment while still following important production-oriented patterns such as private networking, security groups, encrypted storage, and managed credentials.

## Resources Created

* `aws_db_subnet_group`
* `aws_db_instance`

## Current Use Case

In this project, the module creates a private MariaDB RDS instance for the Uptime Kuma environment.

The database is deployed into private data subnets and is not publicly accessible.

## Architecture

The RDS instance is placed in private data subnets inside the VPC.

Access to the database is controlled by a dedicated RDS Security Group. The database only accepts traffic from the EKS worker node security group on the configured database port.

For MariaDB/MySQL, the default port is:

```text
3306
```

## Cost Optimization

The development environment uses a cost-optimized RDS configuration:

* Single-AZ deployment
* Small instance class
* Minimal allocated storage
* Short backup retention
* No read replicas
* No public access
* Deletion protection disabled for easy cleanup

For production, this configuration should be improved with:

* Multi-AZ deployment
* Longer backup retention
* Deletion protection enabled
* More restrictive operational policies
* Monitoring and alerting
* Optional read replicas depending on workload requirements

## Security Notes

The RDS instance is private and is not exposed to the internet.

The master password is not hardcoded in the Terraform code. It is generated by the Secrets Manager module and passed into this module as a sensitive input.

The password is still stored in Terraform state, so the state backend must be secured and must never be committed to Git.

## Git Safety

This module does not contain hardcoded secrets.

Safe to commit:

* `main.tf`
* `variables.tf`
* `outputs.tf`
* `README.md`

Do not commit:

* `terraform.tfstate`
* `terraform.tfstate.backup`
* `.terraform/`
* `*.tfplan`
* real `terraform.tfvars`
* real `backend.tf`
